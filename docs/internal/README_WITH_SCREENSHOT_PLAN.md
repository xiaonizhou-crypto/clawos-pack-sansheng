# README WITH SCREENSHOT PLAN

> 目标：把现有真实截图纳入三省六部项目 README / release 展示层。  
> 原则：不等完美真图，先用真实系统截图撑起首发可信度。

---

## 一、截图使用总策略

当前这批截图不完全是“三省六部古风海报”，
但它们有一个更重要的优点：

> **它们证明这套系统是真的，而且已经有 roster / dashboard / hiring 视图。**

所以 README 展示策略不应该强行把它们当纯视觉海报，
而应该把它们当：
- 产品实机图
- 系统运行图
- 组织能力证明图

---

## 二、推荐选用的三张图

### 图 1：员工列表 / 员工卡片页
#### 作用
作为 README 第一张实机图。

#### 想证明什么
- 你拥有一整套班底
- 不只是名字，而是可直接派活的员工/角色 roster
- 组织感不是嘴上说说

#### 推荐 caption
**你的班底已经到位。**  
直接查看每位成员状态、当前位置与当前任务，并当场布置任务。

#### 建议放置位置
放在 README 开场钩子之后、`朝廷阵容` 之前。

---

### 图 2：后台总览 / 仪表盘页
#### 作用
作为 README 第二张图，用来证明系统成熟度。

#### 想证明什么
- 这不是世界观空壳
- 系统有会话、代理、成本、活跃度等管理能力
- 组织系统真的在运行

#### 推荐 caption
**不是换皮，是可运行的组织系统。**  
从会话、成员到运行成本，这套系统已经具备真实的管理与观测能力。

#### 建议放置位置
放在 `为什么它和普通 AI 不一样` 之后。

---

### 图 3：员工市场 / 招聘页
#### 作用
作为 README 第三张图，用来证明扩编能力。

#### 想证明什么
- 这套组织可以持续扩编
- 不只是固定 roster，而是还能继续招聘 / 增补能力位
- 与 hiring / 组织包叙事天然相连

#### 推荐 caption
**班底不是静态的，还能继续扩编。**  
从员工市场直接招聘新角色、新能力位，把组织系统越补越完整。

#### 建议放置位置
放在 `三档预设怎么选` 之后，或 `深入资源` 之前。

## 四、推荐展示语气

由于截图本身偏现代产品界面，README 的配图说明建议不要过度使用：
- 上朝
- 官员
- 圣旨
- 封驳

而应更多使用这些词：
- 班底
- 成员
- roster
- 组织系统
- 派活
- 扩编
- 招聘

### 这样做的好处
能保留三省六部世界观，
但避免真实截图与文案表达打架。

---

## 五、最终表达建议

推荐保留这条主表达：

> **三省六部是这套组织系统的主题表达。**

但在截图 caption 与实机展示里，多使用：

> **班底 / roster / 组织系统 / 员工市场 / 派活 / 扩编**

这样会更容易让第一次看到的人理解产品价值。

## 七、结论

当前最稳的首发图策略，不是等完美海报，
而是：

> **先用真实系统截图证明这东西能跑、像产品、像组织；再用世界观文案把它变得有记忆点。**
