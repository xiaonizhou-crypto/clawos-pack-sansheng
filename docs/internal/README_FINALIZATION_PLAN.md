# README FINALIZATION PLAN

> 目标：把当前 README 从“内容已很多”收成“首发页节奏清楚、商品感更强”的版本。  
> 原则：删重复、提价值、保实机图、让 CTA 更顺。

---

## 一、当前 README 的主要问题

### 1. 信息已经够多，但有一点散
现在 README 已经有：
- 标题
- slogan
- CTA
- 三张截图
- 阵容表
- 差异说明
- 安装说明
- preset 说明
- 运转逻辑

问题不是不够，而是：

> **用户第一次看时，会花一点时间判断“先看哪”。**

### 2. “朝廷感”和“组织系统感”都在说，节奏还可以更明确
当前 README 已经开始平衡：
- 三省六部世界观
- roster / dashboard / hiring 的现代产品感

这是对的。

但还可以再更清楚地分层：
- 第一屏卖价值
- 中段卖结构与差异
- 后段卖安装与使用

### 3. 阵容表偏早、偏密
角色表很完整，但在第一张实机图后马上上大表，阅读压强会比较高。
