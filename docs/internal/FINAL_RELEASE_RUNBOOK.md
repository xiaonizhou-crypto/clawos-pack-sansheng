# FINAL RELEASE RUNBOOK

> 目标：把三省六部组织包首发动作整理成一份可执行 runbook。  
> 原则：不再泛讲方向，只讲“发的时候按什么顺序做”。

---

## 1. 首发前最终确认

### Step 1：确认仓库名
建议正式首发时使用：
- `clawos-pack-sansheng`

如果暂时来不及改本地目录，也建议在 GitHub 首发仓库名上直接使用该名称。

### Step 2：确认 README 首页
检查：
- 标题是否足够强
- 一句话钩子是否足够直白
- CTA 是否足够靠前
- 阵容表是否清楚
- 安装命令是否显眼

### Step 3：确认三档可跑
再次确认：
- Lite
- Standard
- Full

至少保留最近一次成功验证记录。

## 6. 首发后第一轮观察重点

发出去后，最该观察的不是虚荣指标，而是：

### 1. 用户第一眼能不能懂
- 是否迅速理解这是“AI 朝廷”而不是古风模板

### 2. 用户是否愿意装
- 是否有人直接跑 install
- 是否有人卡在 preset 选择

### 3. 用户是否觉得值
- 是否觉得角色有差异
- 是否觉得组织感明显
- 是否觉得不是普通多 agent 拼盘

---

## 7. 发版后第一轮补强优先级

### 第一优先
- 真 Hero Cover
- 真 Roster Wall
- 真 Preset Comparison

### 第二优先
- README 首屏继续视觉化
- 角色卡图

### 第三优先
- 更细的角色打磨
- Custom 安装
- marketplace 化迁移

---

## 8. Runbook 结论

如果今天开始准备首发，最优顺序是：

1. 定名
2. 定首页
3. 定三张图
4. 定 release 正文
5. 发出第一版

不要再横向开新坑。
首发最重要的是：

> **让用户一眼想装，一装能跑，装完觉得值。**
