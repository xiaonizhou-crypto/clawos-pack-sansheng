# HIGH STAR POLISH NOTES

> 从高星项目常见表现里提炼出的打磨要点，专门用于继续优化三省六部组织包。

## 1. 首页第一屏必须一眼讲清价值
高星项目通常不会让用户先读半天背景再懂产品。

对三省六部来说，第一屏必须立刻讲清：
- 这是什么
- 和普通 AI 有什么不同
- 装上后你会得到什么

当前已具备：
- 强标题
- 一句话定位
- 安装后获得什么

后续还可继续增强：
- 增加更直观的 hero 图
- 增加一句更强的 CTA

---

## 2. 安装路径必须短
高星项目通常都有非常清晰的：
- 安装
- 验证
- 开始用

对三省六部来说，已经有：
- install-pack.sh
- verify-pack.sh
- uninstall-pack.sh

后续可继续增强：
- README 里给一个 "30 秒装上朝廷" 小节
- 把 Lite / Standard / Full 命令放得更显眼

---

## 3. 结构化展示比长篇解释更有转化
高星项目很会用：
- bullets
- 表格
- 分节标题
- roster
- quick comparisons

对三省六部来说，最有效的结构是：
- Hero 介绍
- 角色阵容
- preset 对比
- 安装命令
- 角色卡二跳页

---

## 4. 视觉资源要尽快上首页
高星项目的 README 往往不是纯文字。

对三省六部来说，头像已经到位，下一步应尽快：
- 上 hero cover
- 上 roster wall
- 上 1~3 张角色卡示意图

---

## 5. 文案既要有情绪，也要有功能感
高星项目的文案一般都能同时做到：
- 有记忆点
- 不空
- 有明确使用收益

三省六部也应继续保持：
- 宏大叙事
- 组织感
- 具体收益
- 安装后体验

---

## 6. 最能拉动传播的，不是“功能多”，而是“概念一眼成立”
对三省六部来说，这个概念已经很强：

> 不是安装一个 bot，而是召来一整座替你办事的朝廷。

后续所有 README、宣传图、角色卡，都应该围绕这个核心概念收束。

---

## 7. 下一步最值得继续抛光的点
- hero cover 真图
- README 里更强的 CTA 区块
- 30 秒上手段落
- 角色卡图落地
- Lite / Standard / Full 的一页式对比图
