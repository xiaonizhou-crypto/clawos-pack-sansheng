# THEME PACK PLATFORM STORY

> 目标：把“三省六部只是第一套”这件事讲清楚，避免后续西游 / 后宫 / 仙班等主题看起来只是杂乱换皮。

---

## 核心判断

现在最重要的不是把三省六部单独吹成一个孤立产品，
而是让外界逐渐看懂：

> **你们真正做的是一个可扩展的 AI 组织内核，三省六部只是第一个落地主题包。**

这件事一旦讲清，后面的西游、后宫、仙班，都会从“换皮”升级成：

- 同一个内核的不同世界观版本
- 同一种组织设计能力的多套表达
- 一个 theme pack platform，而不是一个单次灵感项目

---

## 一句话主叙事

### 中文
我们不是在做一个古风 AI 包，而是在做一个可持续扩展的 AI 组织主题平台；三省六部只是第一套已经跑通的皮肤与结构表达。

### English
We are not building a one-off themed AI pack. We are building a reusable AI organization platform, and 三省六部 is simply the first theme pack that has actually landed.

---

## 为什么这件事重要

### 1. 它让三省六部不再像一次性创意
如果只讲三省六部，外界很容易把它理解成：
- 一个很聪明的主题
- 一个有趣的包装
- 一个单独的创意项目

但如果讲“theme pack platform”，理解会变成：
- 你们有底层方法
- 你们有可复用内核
- 你们可以持续扩展新题材

### 2. 它让未来主题都有合理性
西游、后宫、仙班等主题就不会显得像临时脑洞，
而会显得像：
- 同一平台上的新皮肤
- 新的组织编排表达
- 新的目标用户入口

### 3. 它会让 GitHub / 产品传播更像“产品路线”，而不是“设定合集”
这是很关键的。

---

## 应该怎么讲

### 层 1：内核层
先讲：
- roster
- task assignment
- review gate
- execution flow
- expandability
- modular presets

也就是：
**这是一套 AI organization kernel**。

### 层 2：主题层
再讲：
- 三省六部
- 西游
- 后宫
- 仙班

也就是：
**不同文化 / 叙事 / 角色体系，只是这个内核的不同皮肤与世界表达。**
