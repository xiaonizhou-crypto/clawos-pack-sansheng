# Internal Docs

These files are kept for launch planning, messaging, screenshot selection, positioning notes, and release preparation.

They are useful for:
- refining the pack over time
- preserving product and launch thinking
- avoiding redoing the same positioning work later

But they are not the primary public-facing entry point.
