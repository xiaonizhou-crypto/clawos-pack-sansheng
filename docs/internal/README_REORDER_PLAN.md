# README REORDER PLAN

> 目标：把仓库首页调整成更像高星项目与高转化商品页的结构，让用户在最短时间内产生“想装、能装、值得装”的感觉。

---

## 推荐首页结构

### 1. Hero 区
内容：
- 主标题
- 一句话价值钩子
- 次标题
- 主 CTA（安装完整朝廷）
- 次 CTA（查看角色阵容）

目标：
- 10 秒内让用户知道这是什么
- 立刻建立“不是一个 bot，而是一座朝廷”的概念

---

### 2. 朝廷阵容预览
内容：
- 11 角色概览
- 每个角色一句话
- 头像矩阵/角色表

目标：
- 一眼建立阵势感
- 让用户觉得“这真是完整班底，不是口号”

---

### 3. 为什么它不一样
内容：
- 与普通 AI / 普通多 agent 的差异
- 组织感、审核门、调度链、执行链

目标：
- 让用户理解产品价值，不只被设定吸引

---

### 4. 安装后会得到什么
内容：
- Full 母本说明
- Lite / Standard / Full 预设简介

目标：
- 把“完整朝廷先给你，再允许你裁剪”的产品策略说清楚

---

### 5. 30 秒安装
内容：
- 最短命令
- 预设命令
- 验证命令

目标：
- 尽量降低首次行动门槛

---

### 6. 三档预设对比
内容：
- Lite / Standard / Full
- 各自适合谁
- 各自包含哪些角色

目标：
- 降低选择焦虑

---

### 7. 朝廷如何运转
内容：
- 太子 → 中书省 → 门下省 → 尚书省 → 六部执行

目标：
- 让“制度感”变得清楚具体

---

### 8. 适合谁 / 不适合谁
内容：
- 使用画像
- 不适合场景

目标：
- 提升用户自我判断效率

---

### 9. 资源入口
内容：
- `ROLE_ROSTER.md`
- `PRESET_GUIDE.md`
- `30_SECONDS_TO_INSTALL.md`
- `STORE_COPY.md`
- `SCREENSHOT_SCRIPT.md`

目标：
- 给深度用户更多入口，但不打断首页节奏

---

## 结构原则

### 原则 1
先卖概念，再卖结构，再给命令。

### 原则 2
先让用户“想装”，再让用户“知道怎么装”。

### 原则 3
README 不是文档堆砌，而是首页销售页。

---

## 最终目标

用户打开首页后，应该依次经历：
1. 觉得这概念很强
2. 觉得这结构很完整
3. 觉得这安装不难
4. 立刻想试
