# NAME AND POSITIONING SYNC

> 目标：让仓库名、README 表达、产品定位三者保持一致，避免“内容是 Full 母本，名字却像删减版样品”的错位。

---

## 当前问题

当前本地目录仍为：
- `clawos-pack-sansheng-lite`

但内容实际上已经是：
- Full 母本
- Lite / Standard / Full 只是安装预设

所以如果正式首发时还沿用 `lite`，会削弱产品姿态。

---

## 推荐首发名

### 首选
`clawos-pack-sansheng`

### 为什么最适合
- 干净
- 不锁死在某个预设
- 和 Full 母本策略一致
- 后续扩版本、出 release、接 marketplace 都顺

## README 表达建议

如果首发使用 `clawos-pack-sansheng`，README 首页表达应统一：

- 仓库是 **三省六部完整母本**
- 安装支持 Lite / Standard / Full
- 用户可以先装、再裁、再调编制

也就是说，表达核心不能再是“lite 包”，而应是：

> 一整座朝廷的母本，在此仓库中。
