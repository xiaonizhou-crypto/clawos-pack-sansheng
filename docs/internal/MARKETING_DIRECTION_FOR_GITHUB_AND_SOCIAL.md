# MARKETING DIRECTION FOR GITHUB AND SOCIAL

> 目标：明确三省六部项目在 GitHub 与对外传播时，应该优先强调什么、弱化什么。

---

## 总原则

### GitHub 主阵地
优先强调：
- 这是一个可运行的 AI 组织系统
- 支持 roster / assign / review / execute / hire
- 有真实截图
- 有 quickstart
- 有 verify

### 社交传播 / 话题传播
优先强调：
- 不是装一个 bot，而是装一整座替你办事的朝廷
- 三省六部这个钩子为什么让人记得住
- 为什么它比普通多 agent 更有组织感

---

## 一句话区分

### GitHub 版
三省六部是一个可运行的 AI 组织系统，支持 roster、任务派发、审核封驳、执行交付与扩编补位。

### Social 版
不是安装一个 bot，而是召来一整座替你办事的朝廷。

---

## 为什么要这样分

因为高星 GitHub 项目最强的地方，在于：
- 先让人理解
- 再让人记住

而不是反过来。

三省六部如果一开始只讲朝廷世界观，会更容易被当成：
- 古风皮肤
- 角色主题包
- 氛围型包装

但如果先讲清它是：
- 可运行
- 可派活
- 可审核
- 可扩编

再用“三省六部”做记忆点，信任感会高很多。

---

## 对当前项目的执行建议

### GitHub README / Release
继续优先使用：
- roster-overview
- dashboard-overview
- employee-marketplace

### 社媒/口头传播
继续优先使用：
- 朝廷
- 班底
- 上朝
- 替你办事

但不要脱离产品能力去空讲。

---

## 结论

最好的传播路线不是二选一，而是：

> **GitHub 先卖产品，社交传播再放大记忆点。**
