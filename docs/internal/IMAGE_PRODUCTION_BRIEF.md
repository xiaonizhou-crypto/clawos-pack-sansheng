# IMAGE PRODUCTION BRIEF

> 目标：把三省六部首发最重要的三张图，整理成真正可执行的出图 brief。  
> 用途：自己做图、交给设计、交给前端实现展示页，都可以直接照着执行。

---

## 总体要求

### 核心气质
- 豪华
- 秩序
- 官僚中枢感
- 不是古装 cosplay，而是“帝国治理系统”的高级感

### 禁忌
- 不要做成廉价手游海报风
- 不要把信息塞太满
- 不要让 11 个头像挤成杂乱拼图
- 不要只剩古风，没有产品感

### 统一目标
看完图之后，用户心里应自然冒出一句：

> 这不是 AI 模板，这是我能立刻装到自己机器上的一整座朝廷。

---

## 图 1：Hero Cover

### 作用
抓住注意力，建立概念。

### 关键问题
- 这是什么？
- 为什么和普通 AI 完全不同？

### 画面结构
- 中轴排布
- 中间偏前：太子 / 中书省 / 门下省 / 尚书省 / 工部
- 左右两翼：其余角色作阵列支撑
- 背景有殿宇、纹章、秩序感，但不要喧宾夺主

### 文案
**主标题**：三省六部  
**副标题**：不是安装一个 bot，而是召来一整座替你办事的朝廷。  
**辅助说明**：接旨、草拟、审核、派发、执行——让 AI 从各说各话，变成真正有秩序的组织系统。  
**主 CTA**：安装完整朝廷  
**次 CTA**：先看角色阵容

### 输出建议
- 正方形版：适合社媒 / marketplace card
- 横版：适合 README 顶图 / landing page hero

---

## 图 2：Roster Wall

### 作用
建立“完整班底”真实感。

### 关键问题
- 你会得到谁？
- 这些角色是不是各司其职？

### 画面结构
- 11 头像整齐矩阵
- 最好分层：
  - 中枢
  - 六部
  - 翰林院
- 每个头像下只有一个最短标签

### 标签
- 太子：接旨分案
- 中书省：草拟谋划
- 门下省：审核封驳
- 尚书省：统筹派发
- 吏部：编制治理
- 户部：资源轻重
- 礼部：表达呈现
- 兵部：推进督办
- 刑部：风险边界
- 工部：执行交付
- 翰林院：深度研究

### 文案
**总标题**：你的朝廷班底  
**副标题**：不是几个换皮角色，而是一整套真正有分工的官僚组织。

### 输出建议
- 长图版：适合 README / 详情页
- 方图版：适合社媒与分享卡

---

## 图 3：Preset Comparison

### 作用
推动安装决策。

### 关键问题
- 我该装哪个？
- 会不会太复杂？
- 是不是一定得全装？

### 画面结构
- 三列：Lite / Standard / Full
- 每列有简短说明 + 对应角色名单/头像
- Full 明显最豪华，但 Lite 也显得不寒酸

### 文案
**总标题**：先把整座朝廷给你，再由你决定谁留下  
**副标题**：Full 母本，按需裁剪。你可以轻装上朝，也可以完整上朝。

### 每列文案
**Lite**：轻装上朝  
先装核心五核，最快体验朝廷闭环。

**Standard**：常朝阵容  
在中枢之外，补齐内容、资源、推进与风控四部。

**Full**：完整朝廷  
整座朝廷齐备，连编制治理与翰林深研都在位。

### 输出建议
- 横版优先
- 适合 README 中段和产品详情页

## 发布使用顺序
1. Hero Cover
2. Roster Wall
3. Preset Comparison

这个顺序不能乱。

### 原因
- 第一张抓住人
- 第二张建立真实感
- 第三张促成安装动作

---

## 最后判断标准

如果这三张图做完后，用户还不能在 10~20 秒内感受到：
- 这是什么
- 我会得到谁
- 我该怎么装

那就还不够好。
