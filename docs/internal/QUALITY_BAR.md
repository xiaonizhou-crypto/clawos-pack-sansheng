# QUALITY BAR

> 目标：定义三省六部组织包在“可对外发布”之前必须达到的质量标准。  
> 这不是愿景文档，而是发版前的硬门槛。

---

## 1. 产品感门槛

### 必须满足
- 用户打开 README 第一屏，能在 **10 秒内** 明白：
  - 这是什么
  - 和普通 AI 有什么不同
  - 装上后会得到什么
- 用户能在 **30 秒内** 找到安装命令
- 用户能在 **1 分钟内** 理解 Lite / Standard / Full 的区别

### 不满足即不达标
- 文案只剩概念，没有安装路径
- 组织感没有被一眼传达出来
- 角色多但用户不知道为什么要这些角色

---

## 2. 工程门槛

### 必须满足
- `install-pack.sh` 可运行
- `verify-pack.sh` 可运行
- `uninstall-pack.sh` 可运行
- Lite / Standard / Full 三档都至少有一次真实通过记录
- Full 预设能创建完整 roster
- 每个角色对应的必要 skill 都能正确落入 workspace

### 不满足即不达标
- 某一档 preset 只是写了但没跑过
- skill 目录存在但安装不落地
- README 和实际命令不一致

---

## 3. 角色质量门槛

### 必须满足
每个角色至少具备：
- `SOUL.md`
- `USER.md`
- `IDENTITY.md`
- 角色 README

并且：
- `SOUL.md` 有身份、行为准则、边界、气质
- `USER.md` 明确用户和该角色的关系
- `IDENTITY.md` 像一张角色卡，而不是几行参数

### 不满足即不达标
- 角色说话方式没有差异
- 角色只是换名同质化
- 扩展位明显比核心位粗糙太多

---

## 4. 沉浸感门槛

### 必须满足
- 用户能感觉到这是一座朝廷，而不是一组古风命名机器人
- 各部门气质明显不同
- 文案有古风官场气，但不晦涩
- 偶有雅句点染，但不影响理解

### 不满足即不达标
- 角色古风只是外皮
- 文风统一得像同一个人换皮
- 文案一味堆典故，反而影响可读性

---

## 5. 视觉门槛

### 必须满足
- 11 张角色头像全部到位
- avatar 路径已与角色 identity 对齐
- README / roster / screenshot script 能消费这些资产
- 至少有明确的 cover / screenshot 规划

### 下一阶段更优目标
- 真正产出 hero cover
- 产出 roster wall
- 产出 3 张核心角色卡图

---

## 6. 商品表达门槛

### 必须满足
- 有主 slogan
- 有 50 / 120 / 300 字版本介绍
- 有 role roster 总览
- 有 preset guide
- 有 30 秒安装文案
- 有 hero copy block

### 不满足即不达标
- 只能讲工程，不能讲产品
- 只能讲功能，不能讲体验
- 用户看完觉得“有点意思”，但不想立刻装

---

## 7. 真正值得发出去的状态

一版真正值得发出去的三省六部，应该让用户产生这三种明确感受：

### 感受 1：想装
“这不是普通 AI，我想试试。”

### 感受 2：能装
“看起来不复杂，我现在就能装。”

### 感受 3：值装
“装上以后，我真的像拥有了一整座朝廷。”

如果三者缺一，就还不够。

---

## 8. 最终标准

> 不是“做完了就发”，而是“做到了让人一看就想装、装了就想留、留了就觉得值”。

这才配得上对外首发。
